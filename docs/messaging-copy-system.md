# Emerito: Messaging & Copy System

Companion to `positioning-seo-strategy-2026.md`.

**Architecture principle:**
- **Home = the studio.** A gallery of everything Emerito makes, characters, brand content, games, experiences, training series. Broad, confident, no hard sell. Its job: make the range obvious and route each visitor to their door.
- **Landing pages = the sale.** Problem-first, Knowlify-style heroes live here, one per use case × industry. They receive search traffic and ads, and they convert. Training/safety is the *first* landing page vertical, not the studio's identity.

## 2. Landing page hero system (problem-first)

Used on use-case landing pages only, never the home page.

### Rotating H1, training/safety landing page (EN)
1. **Your workers sleep through safety training.**
2. **Your employees forget the course by Friday.**
3. **Your compliance videos get clicked through, not watched.**
4. **Your training competes with phones. And loses.**
5. **Your best procedures live in a manual nobody revisits.**

*(4 and 5 rewritten from earlier draft, same truth, less finger-pointing: the content loses a fair fight, the buyer isn't mocked.)*

### Constant solution line
> We build characters and stories that make training impossible to ignore, and hard to forget.

### Qualifier
> For teams whose safety, training, or culture content can't afford to be skipped.

### CTAs
Primary: **See the proof →** (Working at Heights case) · Secondary: **Book a call**

### ES (landing de formación/seguridad)
1. **Tus trabajadores se duermen en la inducción de seguridad.**
2. **Tu equipo olvida la capacitación antes del viernes.**
3. **Tus videos de cumplimiento se dan clic, no se ven.**
4. **Tu capacitación compite con el celular. Y pierde.**
5. **Tus mejores procedimientos viven en un manual que nadie vuelve a abrir.**

Solución: *Creamos personajes e historias que hacen imposible ignorar la formación, y difícil olvidarla.*

### Rules for writing problem statements
- Second person, present tense, uncomfortable but true. The *content* fails, never the buyer.
- Each statement targets ONE buyer (HSE / L&D / Comms / Brand); each eventually gets its own landing page.
- Blunt beats clever, but blunt ≠ harsh: if a line reads like blame, reframe it as an unfair fight the old format loses.

## 4. Landing-page matrix (the growth engine)

Each cell = one landing page using the §2 formula with a tailored problem statement. Build progressively:

|  | Construction | Manufacturing | Energy/Mining | Logistics | Healthcare |
|---|---|---|---|---|---|
| **Safety training** | ✅ build 1st (case-study match) | 2nd | later | later | later |
| **Onboarding** | later | later | later | later | later |
| **Culture change** | later | later | later | later | later |

Other verticals get their own landing pages outside the matrix as demand appears: brand characters, games & experiences, activations.

Template per page: problem H1 → solution line → video proof → 1 case result → format options → LMS/SCORM note → pricing anchor → single CTA. ~600–900 words targeting "[use case] videos for [industry]".

## 5. Productized offer (the "automatic sales" layer, no app required)
- **Named packages, fixed scope & price**: *Character Pilot* (1 character + 1 episode) · *Training Series* (character + 4–6 episodes) · *Brand Character Program* (IP + campaign kit).
- **Transparent pricing ranges** on landing pages (top BOFU converter).
- **ROI calculator** (later): cost of failed training/incidents vs. content that sticks.
- Buying feels self-serve; delivery stays studio. If one package dominates, that's the signal for a future product, decided by data, not by redesign.
